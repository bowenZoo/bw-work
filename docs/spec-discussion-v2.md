# 讨论系统 V2 规格文档

> 版本：v2.0
> 日期：2026-02-06
> 状态：待评审

## 1. 概述

重构多 Agent 讨论系统，实现：
- 并行发言（被点名者同时思考回答）
- 全局单例（所有用户看同一个讨论）
- 精简对话（像人说话，简短多轮）
- 讨论恢复（从历史继续）
- 议题展示（前端显示议题+附件预览）

## 2. 核心改动

### 2.1 并行发言机制

**当前**：顺序执行，A → B → C → 主策划总结

**目标**：主策划点名后，被点名者并行回答

```
主策划开场（提问 A 和 B）
    ├── A 思考并回答 ──┐
    └── B 思考并回答 ──┴── 主策划总结
```

**技术方案**：
- 解析主策划的开场/总结，识别被点名角色
- 使用 `asyncio.gather()` 并行调用被点名的 Agent
- 未被点名的 Agent 本轮静默
- WebSocket 实时广播各 Agent 状态（thinking/speaking）

**实现要点**：
```python
# 识别被点名角色（从主策划输出解析）
mentioned_roles = parse_mentioned_roles(lead_planner_output)

# 并行调用
tasks = [agent.respond(context) for agent in agents if agent.role in mentioned_roles]
responses = await asyncio.gather(*tasks)
```

### 2.2 全局单例讨论

**当前**：每个用户可创建自己的讨论

**目标**：全站只有一个活跃讨论，所有用户观看/参与

**技术方案**：
- 后端维护全局 `current_discussion_id`
- 新用户加入时获取当前讨论状态和历史消息
- WebSocket 广播给所有连接的客户端（不按 discussion_id 分组）
- 讨论结束后，任何用户可发起新讨论

**API 改动**：
```
GET  /api/discussions/current     # 获取当前活跃讨论
POST /api/discussions/current     # 创建新讨论（替换当前）
POST /api/discussions/current/join # 加入当前讨论（获取历史）
```

**WebSocket 改动**：
- 移除 `discussion_id` 路径参数
- 所有客户端连接同一个端点：`ws://host/ws/discussion`
- 新连接时推送当前讨论状态 + 历史消息

### 2.3 用户参与机制

**场景**：观看讨论的用户可以发表意见，参与讨论

**交互流程**：
```
┌─────────────────────────────────────────────┐
│ 讨论进行中...                                │
│                                             │
│ 👑 主策划: 关于付费模式，大家怎么看？         │
│ 👤 数值策划: 我建议采用...                   │
│                                             │
│ ────────────────────────────────────────    │
│ 💬 [用户输入框] "我觉得付费点太多了"  [发送]  │
│ ────────────────────────────────────────    │
│                                             │
│ 🙋 用户: 我觉得付费点太多了                  │
│ 👑 主策划: 感谢反馈！玩家代言人，你怎么看     │
│           用户提到的付费点问题？              │
│ 👤 玩家代言人: 用户说得对，从玩家角度...      │
└─────────────────────────────────────────────┘
```

**技术方案**：
1. **前端**：讨论页底部添加输入框（仅讨论进行中时可用）
2. **后端**：调用现有 `inject_message()` API
3. **主策划响应**：收到用户消息后，主策划需要：
   - 感谢/回应用户
   - 将问题转给相关角色
   - 或解答用户疑问

**Prompt 增强**（主策划）：
```
当收到用户消息时：
1. 先简短回应用户（"感谢反馈"/"好问题"）
2. 判断是否需要其他角色回答
3. 如需要，点名相关角色回应用户问题
4. 如果是简单问题，直接回答
```

**限制**：
- 用户每分钟最多发送 2 条消息（防刷屏）
- 消息长度限制 200 字
- 讨论暂停/结束时输入框禁用

### 2.4 精简对话

**当前**：Agent 输出冗长，动辄千字

**目标**：像人说话，简短、聚焦、多轮

**技术方案**：修改 Agent prompt

```yaml
# 每个 Agent 的 backstory/goal 中添加约束
constraints:
  - 每次发言不超过 200 字
  - 直击要点，不说废话
  - 有不同意见直接说，不绕弯
  - 不重复别人说过的内容
  - 如需详述，等主策划追问
```

**主策划控场**：
- 主策划决定每轮讨论的焦点
- 主策划决定何时需要深入、何时进入下一话题
- 主策划决定讨论是否充分

### 2.5 议程管理（动态讨论列表）

**需求**：
- 显示总共要讨论的观点/议题
- 显示当前正在讨论的观点
- 主策划每轮可动态调整议程
- 讨论完一个观点后生成小结文档

**布局设计**：
```
┌─────────────────────────────────────────────────────────┐
│ 📋 讨论议程                              [展开/收起]     │
├─────────────────────────────────────────────────────────┤
│ ✅ 1. 核心玩法定义           [查看小结]                 │
│ 🔵 2. 付费模式设计  ← 当前                              │
│ ⬜ 3. 数值平衡方案                                      │
│ ⬜ 4. 新手引导流程                                      │
│ ➕ [主策划可添加新议题]                                  │
└─────────────────────────────────────────────────────────┘
```

**状态说明**：
- ⬜ 待讨论
- 🔵 进行中
- ✅ 已完成（可点击查看小结）
- ❌ 跳过

**主策划控制**：
- 主策划开场时生成初始议程
- 每轮总结时可：
  - 标记当前议题完成
  - 调整下一个要讨论的议题
  - 添加新发现的议题
  - 跳过某个议题

**议题小结生成**：
讨论完一个议题后，自动生成小结文档：
```markdown

## 讨论结论
- 采用 Battle Pass + 外观商城模式
- 不做数值付费

## 各方观点
- 系统策划：技术实现方案...
- 数值策划：定价建议...
- 玩家代言人：用户接受度...

## 遗留问题
- 首充礼包定价待定

## 相关讨论
- [点击查看完整讨论记录]
```

**点击预览**：
- 议题小结支持 Modal 预览
- 可下载为 Markdown/PDF
- 可分享链接

### 2.6 讨论恢复

**当前**：有 `/api/discussions/{id}/continue` API，前端未完善

**目标**：从历史记录中选择讨论，继续讨论

**技术方案**：
- 历史列表页添加"继续讨论"按钮
- 继续时，加载原讨论的上下文作为背景
- 用户输入追加问题/方向
- 创建新讨论，引用原讨论

**前端改动**：
- 历史列表项添加"继续"操作
- 继续对话框：显示原议题 + 追加输入框
- 进入讨论页面，显示"续前讨论"标识

## 3. 数据模型变更

### 3.1 全局讨论状态

```python
# backend/src/api/routes/discussion.py

# 全局活跃讨论（单例）
_current_discussion: DiscussionState | None = None
_current_discussion_lock = threading.Lock()

def get_current_discussion() -> DiscussionState | None:
    with _current_discussion_lock:
        return _current_discussion

def set_current_discussion(discussion: DiscussionState) -> None:
    with _current_discussion_lock:
        global _current_discussion
        _current_discussion = discussion
```

### 3.2 讨论消息增强

```python
class Message:
    id: str

    mentioned_by: str | None = None  # 被谁点名
```

## 6. 风险与注意事项

1. **并行冲突**：多个 Agent 同时输出时，前端消息顺序可能错乱
   - 方案：消息带序号，前端排序

2. **全局单例竞争**：多用户同时发起新讨论
   - 方案：加锁，先到先得，其他用户收到"讨论已开始"提示

3. **精简过度**：Agent 回答太短失去价值
   - 方案：允许主策划追问"请详细说明"

4. **恢复上下文过长**：历史讨论内容太多
   - 方案：只加载摘要 + 最后 N 条消息

## 7. 验收标准

- [ ] 可从历史记录继续讨论
- [ ] 讨论开始前显示议题和附件
